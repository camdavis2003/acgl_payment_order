Getting Started and Navigation

Open in User Guide

ACGL FMS is a year-based financial workflow system for Payment Orders, Budget, Ledger, Income, Money Transfers, Archive, and administration. Nearly every page uses the same shell: a left navigation tree, a top header, and a main work area.

How to move around the app

  1. Click the top-left menu icon to open the left navigation.
  2. Expand year-based branches (Budget, Payment Orders, Ledger).
  3. Select the exact year and sub-page you need.

What each area is for

  • Payment Order Request: create a new Payment Order.
  • Payment Orders: review, edit, track, filter, approve, and delete existing orders for a year.
  • Budget: manage account lines, anticipated amounts, and balances.
  • Ledger: see income and Payment Order activity together for a year.
  • Money Transfers: group eligible income rows into formal transfer records.
  • Archive: open historical years without changing the active year.
  • Admin Settings: manage users, numbering, notifications, backups, and audit information.

How year-based pages work

Most financial pages are scoped by year. That means the same module can look different depending on the year you opened. Before editing records, confirm you are working in the correct year branch.

Dark mode

Use the dark mode switch in the header to change the visual theme. The preference is stored in local browser storage and is reused when you reopen the app.

If you do not see a module, your account likely does not have that module permission.

For complete policy and process detail, open User Guide.

Permissions and Page Visibility

Open in User Guide

The left navigation only shows modules your account is allowed to access. If a page is missing, the most common reason is that your user account does not have the required permission level.

Typical access levels

  • Read: you can view records only.
  • Write: you can update existing records.
  • Create: you can create new records in that module.
  • Delete: you can remove records where allowed.
  • Full: you have unrestricted access for that module.

Common visibility questions

  • Why can I not see Archive? Your account does not currently include archive access.
  • Why can I open a page but not edit? You likely have read-only access.
  • Why can I not add a new row? The module may allow viewing but not creation.
  • Why can another user see more menu items? Their account has broader permissions configured in Admin Settings.
Help and User Guide follow the same visibility pattern. If the User Guide is available to you, Help should be available too.

Sign In, Session, and Access

Open in User Guide

How to sign in

  1. Click Sign in in the top-right area.
  2. Enter username and password.
  3. Submit and wait for the session to start.

What happens after sign-in

  • The application loads your page permissions and updates the navigation tree.
  • The app remembers the current browser session until timeout, logout, or session expiration.
  • In shared WordPress mode, authentication is validated against the backend and audit events can be recorded.

If sign-in fails

  • Re-check username spelling and case.
  • Re-check password and keyboard layout.
  • Multiple failed attempts can trigger a temporary lock-out timer.

Session questions

  • Why was I signed out? The session may have expired after inactivity.
  • Why did my menu change after signing in? The menu is rebuilt from your user permissions.
  • Where can admins review access events? In Admin Settings under Audit Log.

Sign-in success/failure and lock-out events are recorded in Audit Log under Admin Settings.

Payment Orders: Create, Edit, and Track

Open in User Guide

Create a new Payment Order

  1. Open Payment Order Request.
  2. Complete required fields: Date, Amount (EUR and/or USD), Budget Nr., Payable To, Purpose, Source.
  3. Optionally complete IBAN/BIC and routing via the With field.
  4. Download PDF and submit.

What the main fields mean

  • Date: the request date for the PO.
  • EUR / USD: the requested amount in one or both currencies.
  • Budget Nr.: the budget account code that the expense belongs to.
  • Payable To: the person or entity to be paid.
  • Purpose: a short description of what is being paid.
  • Source: where the funding is expected to come from.
  • With: who currently holds or routes the PO during review.

Before you submit

  • Verify all required fields are complete.
  • Confirm the Budget Nr. is the correct account.
  • If banking data is required, confirm IBAN and BIC validation passes.
  • Generate the PDF if your process requires saving the formal document before submission.

Find and update existing orders

  • Go to Payment Orders and choose year.
  • Use global table search and status filter.
  • Use row actions to view PDF, edit, or delete.

What to do if a row is highlighted

  • Dark highlighted row: required information is missing. Open edit and complete the missing data.
  • Yellow highlighted row: the PO is in Approved or GT-review state and usually needs treasurer attention.

Common questions

  • Can I change the PO number? No. It is normally generated by the configured numbering rules.
  • Why can I not delete a PO? Your account may not include delete access.
  • Why is my PO not appearing in the year I expected? Confirm the record date and the selected year page.
Yellow rows indicate Approved or GT-Review work that usually needs immediate treasurer follow-through.

Itemize Payment Order

Open in User Guide

Use Itemize when one PO contains multiple expenses (for example, hotel, transport, meals).

  1. Click EUR or USD amount field on the request form.
  2. Add line items and optional mileage items.
  3. Confirm running totals.
  4. Save items to return totals to the request form.

When to use itemization

  • One trip includes several receipts.
  • A single reimbursement spans several budget-relevant expense types.
  • You need a transparent breakdown for later review or audit.

Best practice

  • Use clear item titles.
  • Check both per-item amounts and total amount.
  • Save items before leaving the itemize page so totals return to the form.

For currency conversion support, keep required source evidence (OANDA conversion output or relevant bank statement).

Payment Order Status and Routing

Open in User Guide

Status meanings

  • Submitted: awaiting review.
  • Review: GS and/or GM review in progress.
  • Returned: sent back for corrections.
  • Rejected: declined; not processed.
  • Approved: authorized for payment.
  • Paid: completed and closed.

How status affects workflow

  • Submitted items are waiting for initial review.
  • Review means GS or GM is actively evaluating the request.
  • Returned means the requestor must correct or complete the record and then resubmit if required by the process.
  • Approved usually means the budget impact has been recognized and the record is ready for treasurer follow-through.
  • Paid means execution is complete and the item is closed.

With field options

The With field tracks current custodian: Requestor, Grand Secretary, Grand Master, Grand Treasurer, or Archives.

Common questions

  • Why was my PO returned? It may be missing documentation, contain an incorrect budget number, or need corrections to payee/purpose details.
  • When does the budget update? Approved POs can affect the budget automatically.
  • Why is the row yellow? The record is Approved or GT-review and requires action.

Reconciliation

Open in User Guide

Reconciliation is for entries that must be validated before moving into the main Payment Orders table.

  1. Open Payment Orders -> Year -> Reconciliation.
  2. Verify entry details.
  3. Click Reconcile to move entry into main list.

When reconciliation is used

  • Imported or migrated records.
  • Administrative corrections entered outside the request form.
  • Entries that must be confirmed before joining the main list.

Before reconciling

  • Check payee, purpose, amount, source, and year.
  • Confirm the entry should live in that year's PO list.
  • Make sure any required supporting information is already present.

Budget and Dashboard

Open in User Guide

Budget table basics

  • Code and Description identify each account line.
  • Anticipated is planned amount.
  • Approved is auto-calculated from approved POs.
  • Receipts and Expenditures track actual movement.
  • Balance is computed based on account rules.

Common actions

  • Save Budget to persist manual edits.
  • Set Active Budget to make that year primary in nav.
  • Use checksums to verify data integrity.

How to work safely in Budget

  1. Confirm the selected year first.
  2. Update only the intended account lines.
  3. Save changes explicitly.
  4. Re-check balances and checksum rows after edits if those controls are relevant to your workflow.

How Budget relates to Payment Orders

Payment Orders and Budget are connected. Approved Payment Orders can update budget totals automatically, so incorrect budget numbers on POs lead to incorrect budget reporting.

Dashboard

Dashboard shows Expenditures versus Balance charts with search by account code or description.

Use Dashboard when you want a quick visual review. Use the full Budget page when you need account-level editing and detailed table work.

Ledger and Income Pages

Open in User Guide

Ledger combines income entries and PO activity into a single year-based view, with subtotal visibility and state highlighting.

What Ledger is best for

  • Reviewing income and outflow together in one place.
  • Checking whether a PO has moved into paid workflow.
  • Reviewing MT assignments and pending highlighted items.

BankEUR

  • Add entries with date, description, amount, Budget Nr.
  • Rows missing Budget Nr. are highlighted for follow-up.

Missing-budget income rows should be corrected promptly because they affect reporting and may delay Money Transfer work.

wiseEUR and wiseUSD

  • Import from seed-compatible CSV format.
  • Review incoming, outgoing, and net totals in footer rows.

Common questions

  • Why is a Ledger row yellow? It is typically an Approved or GT-review PO.
  • Why is an income row pale yellow? It may be eligible for a Money Transfer but not yet assigned.
  • Why is a BankEUR row dark/highlighted? It is missing a Budget Nr.

Money Transfers

Open in User Guide

Create a transfer

  1. Open Money Transfers -> Year -> New M.T.
  2. Set MT date.
  3. Select eligible unassigned positive BankEUR rows.
  4. Add comments and save.

How selection works

Money Transfers should be built from eligible, unassigned, positive BankEUR income rows. The linkage is explicit, meaning the user chooses the rows to assign rather than relying on a date-range guess.

Verification flow

Transfer records proceed through GS verification and GT verification. Notifications can be sent at each stage when configured.

Common questions

  • Why is an income row not available for selection? It may already be assigned or may not be eligible.
  • Where do I see the MT number later? It appears on linked records in relevant views such as Ledger.

Archive and Historical Years

Open in User Guide

Archive provides historical year links grouped by module so you can browse prior data without changing active-year configuration.

When to use Archive

  • Review prior year Payment Orders.
  • Inspect old budget tables and ledger data.
  • Check historical Money Transfers and related records.

If Archive is not visible in navigation, your account does not currently include archive access.

Admin Settings

Open in User Guide

Admin Settings controls users, numbering, branding assets, email templates, backups, and audit visibility.

Key areas

  • User Roles: create/update account permissions.
  • PO and MT Numbering: sequence settings.
  • GL Information: logo and signature for PDFs.
  • Email Notifications: recipients and templates.
  • Audit Log: read-only security and action events.
  • Backlog: lightweight internal task tracking.

When to use each area

  • User Roles: when a user needs access added, removed, or corrected.
  • PO and MT Numbering: when numbering rules or counters need to change.
  • GL Information: when PDF branding assets are outdated.
  • Email Notifications: when recipients or templates must be updated.
  • Backup: before major admin changes or at regular checkpoints.

Notifications and Alerts

Open in User Guide

The app can send email notifications for key workflow events when notification settings are configured in shared WordPress mode.

Typical events

  • New Payment Order submitted.
  • Grand Secretary review needed.
  • Grand Master review needed.
  • Grand Treasurer processing needed.
  • Money Transfer GS or GT verification steps.
  • Budget updates or backlog item creation where configured.

What admins usually manage

  • Recipient addresses.
  • Subject and body templates.
  • Which events are enabled.
If an expected email does not arrive, first confirm the event is enabled and the configured recipients are correct.

Backup and Restore

Open in User Guide

WordPress backup

  1. Select budget year.
  2. Create backup snapshot.
  3. Download or restore from backup card.

Cloud backup (Google Drive)

  1. Configure OAuth client in plugin settings.
  2. Authorize Google account.
  3. Create cloud backup and restore when needed.

When to take a backup

  • Before major admin changes.
  • Before restoring older data.
  • Before testing structure or numbering changes.
  • As part of routine operational housekeeping.

Restore caution

Restoring data can replace current records depending on the backup workflow being used. Confirm the selected backup and year before continuing.

Keep at least one downloaded JSON backup outside WordPress for disaster recovery.

Troubleshooting and Common Problems

I cannot see a page in the menu

  • Confirm you are signed in.
  • Confirm your account includes that module permission.
  • If another user can see it and you cannot, ask an admin to review your permissions.

I can see a page but cannot change anything

  • You may only have read access.
  • The specific action may require create, delete, or full access.

A Payment Order row is highlighted dark

The record likely has missing required data. Open edit and check core fields such as amount, budget number, payee, source, and purpose.

A row is highlighted yellow

The record is in an approval/treasurer-related state and usually needs follow-up rather than correction.

Help search is not finding what I typed

  • Try shorter keywords.
  • Search by module name such as payment, budget, ledger, archive, backup, or transfer.
  • Search by action such as approve, reconcile, restore, create, delete, or sign in.
