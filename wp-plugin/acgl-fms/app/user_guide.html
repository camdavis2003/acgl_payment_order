1. Application Overview

Open in Help Center

The ACGL Financial Management System (FMS) is a browser-based financial workflow application for the Ancient, Accepted, and Consecrated Order of the Grand Lodge (ACGL). It manages the full lifecycle of Payment Orders and tracks annual budget activity across multiple account types and currencies.

The system can operate in two modes:

  • Standalone mode — data is stored in the browser's local storage. Suitable for development or offline use.
  • WordPress shared mode — data is stored on a WordPress REST API backend, enabling multi-user shared access. This is the standard production mode.

Key capabilities include: Payment Order request & approval workflow, budget tracking, multi-account income ledger (BankEUR, wiseEUR, wiseUSD), Money Transfer management, archive access, email notifications, and Google Drive cloud backup.

2. Navigation & Layout

The application uses a consistent shell layout across all pages:

  • Left sidebar (Navigation Tree) — click the hamburger icon (☰) in the top-left to open or close the sidebar. The tree lists all modules available to the signed-in user. Year-specific sub-pages (Budget, Payment Orders, Ledger, etc.) appear as expandable branches.
  • Header — contains the page title, the ACGL seal logo, a dark-mode toggle, and the Sign In / user button.
  • Main content area — tables, forms, and cards relevant to each module.

Top-level navigation items

  • Payment Order Request (home)
  • Budget (with Dashboard sub-page)
  • Payment Orders (per budget year, with Reconciliation sub-page)
  • Ledger (per budget year, with BankEUR / wiseEUR / wiseUSD sub-pages)
  • Money Transfers
  • Archive
  • Admin Settings (admin / privileged users only)

Dark Mode

Toggle dark mode using the switch in the top-right corner of any page. The preference is persisted in browser storage.

3. Signing In

Open in Help Center

Click the Sign in button in the header. Enter your username and password, then click Sign In.

  • Sessions are maintained in browser session storage.
  • After a period of inactivity the session expires automatically and you will see the sign-in prompt again.
  • All sign-in events (successful, failed, and lock-outs) are recorded in the Audit Log (Admin Settings → Audit Log).
  • Repeated failed attempts trigger a temporary lock-out with a countdown timer.
Tip: If you share the FMS link with another user, the WordPress REST URL and nonce are automatically appended to the URL so they land in the correct shared-storage context.

4. User Roles & Permissions

Open in Help Center

Permissions are assigned per-module to each user by an administrator in Admin Settings → User Roles. There is no fixed named role (e.g. "Treasurer"); instead, each user receives a combination of module-level access levels.

Access Levels

Each module can be granted one of the following levels:

  • none — module hidden; access blocked.
  • read — view only; no create, edit, or delete.
  • write — view and edit existing records.
  • create — view, edit, and create new records.
  • delete — view, edit, create, and delete records.
  • full — unrestricted access including all administrative actions.

5. Payment Orders

Open in Help Center

Payment Orders (POs) are the core financial document of the FMS. A PO represents a request for reimbursement or expenditure authorisation. All POs are year-scoped.

Submitting a New Payment Order

  1. Navigate to Payment Order Request (the home page).
  2. The Payment Order Nr. is generated automatically based on the numbering scheme in Settings and cannot be edited.
  3. Fill in the required fields (marked with *):
    • Date — date of the request.
    • Euro (€) / USD ($) — amount(s) requested. Click either field to open the Itemize panel and break the total into individual expense lines.
    • Budget Number — the 4-digit account code.
    • Payable To — name of the payee.
    • Purpose — brief description of the expense.
    • Source — funding source (Commerzbank, wiseEUR, wiseUSD, or Form Submission).
  4. Optionally complete supplementary fields: IBAN/BIC (validated automatically), With (routing, e.g. Grand Secretary), and any notes.
  5. Click Download PDF to generate and save the official PDF document for the PO, then click Submit to save the record.

Payment Orders List

Navigate to Payment Orders → [Year] to see all submitted POs for that year. The list is sortable by clicking column headers and supports full-text search. Use the Status dropdown filter to narrow by status.

Each row shows: PO Number, Date, Payable To, Amount, Budget Number, Source, With, Status, and action buttons (View PDF / Edit / Delete).

6. Itemizing a Payment Order

Open in Help Center

When a payment order covers multiple expense categories (e.g. flights, hotel, meals), click the Euro or USD amount field on the Request Form to open the Itemize Payment Order page.

  • Click Add Item to enter a line item with a title, euro amount, and/or USD amount.
  • Click Add Mileage to log a mileage expense; the kilometre rate is applied automatically.
  • The running totals for EUR and USD update automatically as items are added or edited.
  • Click Save Items to write the totals back to the request form.
Currency conversion: If receipts are in a different currency, you must attach either an OANDA Currency Converter PDF for the payment date or a bank statement in the submitted currency.

Status Descriptions

  • Submitted — the PO has been submitted by the requestor and is awaiting initial review.
  • Review — the PO is under active review by the Grand Secretary and/or Grand Master.
  • Returned — the PO was returned to the requestor for corrections or additional documentation.
  • Rejected — the PO was declined and will not be processed.
  • Approved — the PO has been authorised and is ready for the Grand Treasurer to execute payment. Approved rows are highlighted in yellow in the table.
  • Paid — payment has been executed and the PO is closed.

Budget Impact

When a PO is set to Approved, its amount is automatically deducted from the relevant budget line (OUT account) for that year. This keeps the Budget table current without manual entry. Paid POs are also reflected in the Ledger.

Routing ("With") Field

The With field records the current custodian of the PO during the review process. Options are:

  • Requestor
  • Grand Secretary
  • Grand Master
  • Grand Treasurer
  • Archives

Email Notifications Triggered

  • New Payment Order — fires when a PO is submitted.
  • Grand Secretary Review — fires when status moves to Review.
  • Grand Master Review — fires when escalated to GM.
  • Grand Treasurer Processing — fires when Approved.

8. Reconciliation

Open in Help Center

The Reconciliation page (Payment Orders → [Year] → Reconciliation) holds Payment Order entries that were created outside the normal submission flow (e.g. imported records or administrative corrections) and need to be verified before joining the main Payment Orders list.

  1. Review each entry and confirm the details are correct.
  2. Click Reconcile to move the entry into the main Payment Orders list for that year.
  3. Click the back link to return to the Payment Orders list after reconciliation.

9. Budget

Open in Help Center

The Budget page (Budget → [Year]) presents the annual budget table with IN (income/receipts) and OUT (expenditure) accounts. It is the financial plan for the year.

Columns

  • Code — 4-digit account code.
  • Description — account name.
  • Anticipated — the planned budget amount.
  • Approved — total of Approved Payment Orders charged to this account (auto-calculated).
  • Receipts — income received against this account.
  • Expenditures — actual payments made.
  • Balance — computed from the above columns.

Actions

  • Save Budget — persists any manual edits to the budget table.
  • Set Active Budget — marks this year as the active budget. The sidebar "Budget" link will open this year's dashboard until deactivated.
  • Checksums toggle — reveals checksum rows comparing two data sources to verify integrity.

Budget Number Tooltips

Throughout the app, hovering over a budget number (e.g. 2200 – Per Diem and Travel Expenses) displays a tooltip with the full account description.

Balance Calculation Logic

For most OUT accounts: Balance = Anticipated − Receipts − Expenditures. For specific income-linked accounts (New Lodge Petitions, Grand Lodge Charity, Grand Master's Charity): Balance = Approved + Receipts − Expenditures.

10. Budget Dashboard

Accessed via Budget → [Year] → Dashboard (or the sidebar "Budget" link when an active budget is set). Displays bar charts of Expenditures vs Balance for each OUT account, grouped into Anticipated and Actual sections.

  • Use the search box to filter charts by account code or description.
  • Chart colours: ■ Orange = Expenditures, ■ Blue = Balance.
  • Click ← Back to [Year] Budget to return to the full table.

11. Ledger

Open in Help Center

The Ledger (Ledger → [Year]) is a consolidated view that combines all income entries (BankEUR) and Approved/Paid Payment Orders for the selected year on a single scrollable table.

  • Income rows are shown alongside PO rows, giving a running picture of cash in vs cash out.
  • Approved / GT-Review rows are highlighted in yellow (see Color Highlighting Guide).
  • Income rows that are eligible for a Money Transfer but have no MT number assigned are highlighted in a pale yellow (#fff6bf) to flag them as pending.
  • The footer row shows running totals for income and expenditure columns.

Sub-ledger Links

Within the Ledger page, buttons link to the individual account sub-pages:

  • BankEUR — Commerzbank EUR transactions.
  • wiseEUR — Wise EUR transactions.
  • wiseUSD — Wise USD transactions.

12. BankEUR Income

Accessed via Ledger → [Year] → BankEUR. Lists Commerzbank EUR transactions for the selected year.

Adding an Entry

  1. Click Add Entry.
  2. Enter the date, description, amount, and optionally a budget number.
  3. Save.

Row Highlighting

Income rows missing a Budget Number are highlighted in the primary colour (dark background) to indicate they require attention. Assign a budget number via the edit action.

Note: Only unassigned positive income entries are eligible to be linked to a Money Transfer. The linkage is explicit (user selects the entries) — date-range filtering is not used.

13. wiseEUR & wiseUSD

Accessed from Ledger → [Year]. These pages list transactions processed through the Wise platform in EUR and USD respectively.

  • Transaction data can be imported from a CSV file (the seeded CSV format is wise_eur_YYYY_seed.csv / wise_usd_YYYY_seed.csv).
  • The Received From column is truncated for display but the full name appears on hover.
  • Totals (incoming, outgoing, net) are shown in the footer row.

14. Money Transfers

Open in Help Center

Money Transfers (MTs) record the transfer of funds from one account to another (e.g. wiseEUR to Commerzbank). Each MT groups one or more BankEUR income entries and records the transfer details.

Money Transfers List

Navigate to Money Transfers → [Year] to see all MTs. Each row shows: MT Number, Date, Comments, total EUR/USD amounts, GS-verified status, GT-verified status, and actions.

Creating a Money Transfer

  1. Click New M.T. to open the Money Transfer builder page.
  2. Set the MT Date using the date picker.
  3. The table shows all unassigned positive BankEUR income entries for the year. Select the entries to include by checking their checkboxes.
  4. Add any comments and click Save.

Verification Workflow

→

→

  • Email notifications are sent at each verification step (see Email Notifications).
  • Once income entries are assigned to a MT, they are removed from the eligible pool and their MT number is displayed in the Ledger.

15. Archive

Open in Help Center

The Archive page (Archive in the sidebar) provides links to historical years' data, grouped by functional area (Budget, Payment Orders, Ledger, Money Transfers, etc.).

  • Year links navigate directly to the year-scoped version of each module.
  • Access is controlled by the Archive permission key. Users without this permission will not see the Archive link in the sidebar.

Open in Help Center

The FMS can send automatic email notifications when key events occur. Notifications are only active in WordPress shared mode and require configuration in Admin Settings → Email Notifications.

Available Notification Events

Event Triggered When Default Subject
New Payment Order A PO is submitted. [ACGL FMS] New Payment Order {{paymentOrderNo}}
Grand Secretary Review PO status set to Review. [ACGL FMS] Payment Order Awaiting GS Review
Grand Master Review PO escalated to GM. [ACGL FMS] Payment Order Awaiting GM Review
Grand Treasurer Processing PO status set to Approved. [ACGL FMS] Payment Order Approved for GT Processing

New wiseUSD A wiseUSD entry is added. [ACGL FMS] New wiseUSD Entry

Money Transfer GT Verification MT marked as GS Verified. [ACGL FMS] Money Transfer GS Verified
New Backlog A backlog item is created. [ACGL FMS] New Backlog Item

Template Variables

Email subjects and bodies support placeholders that are replaced automatically at send time. Supported placeholders currently used by the built-in templates include:

  • {{paymentOrderNo}} — the PO number.
  • {{paymentOrderLink}} — link back to the PO in the FMS.
  • {{year}} — the budget year.
  • {{createdAt}} — creation timestamp.
  • {{id}} — record ID (used in New Payment Order).
  • {{date}} — entry or event date.
  • {{description}} — transaction/entry description.
  • {{amount}} — amount value.
  • {{party}} — sender/party label (wise entries).
  • {{comments}} — money transfer comments.
  • {{directLink}} — direct deep-link to the record.
  • {{moneyTransferNo}} — the MT number.
  • {{refNo}} — backlog reference number.
  • {{subject}} — backlog subject/title.
  • {{priority}} — backlog priority level.
  • {{createdBy}} — backlog creator name.
  • {{user}} — the username of the person who triggered the event.

19. Backup & Restore

Open in Help Center

The FMS supports three backup mechanisms, all accessible from Admin Settings → Backup.

WordPress Backup (per year)

  1. Select the budget year to back up.
  2. Click Create Backup.
  3. Download the JSON file or store it in WordPress.
  4. To restore, click Restore on a listed backup card and confirm.

Cloud Backup (Google Drive)

Cloud backup stores a single all-years JSON snapshot in Google Drive. It must be enabled first by completing the Google OAuth setup:

  1. In WordPress admin, open ACGL FMS settings and enter your Google OAuth Client ID and Client Secret.
  2. Save, then click the Google authorization button and complete consent.
  3. Return to the Backup section and click Cloud Backup.
  4. Use Download or Restore on a cloud backup entry as needed.
Note: Cloud backup stores one all-years file. WordPress backup cards remain year-based.

Local File Export

Any backup can be downloaded as a JSON file to your device for offline safekeeping.

20. Backlog

The Backlog section (Admin Settings → Backlog) is a lightweight task tracker for the Grand Lodge team. It is not a financial record.

  • Click Add Item to create a new backlog item with a subject, priority, and description.
  • Items can be searched, edited, and marked as archived.
  • Click Archive to toggle the archived items panel.
  • Creating a backlog item triggers the New Backlog email notification if configured.
